## Zettel

**Главная мысль:** в GNU/Linux файл - это не только "документ на диске", а универсальная абстракция доступа к данным и объектам ОС; тип файла, inode, имя в каталоге, права, владелец и ссылки вместе определяют, как объект существует и администрируется.

## Развернутый ответ

В Linux все действия с большим числом объектов выполняются через универсальный интерфейс файловых дескрипторов. **Файловый дескриптор** - это идентификатор открытого потока ввода-вывода. Через него процесс работает с объектом: открывает, читает, пишет и закрывает.

Файл в Linux нужно понимать в двух слоях:

- **имя в каталоге** - запись, которая связывает строковое имя с inode;
- **inode** - индексный дескриптор, где хранятся метаданные и ссылки на данные файла.

Важный вывод: сам файл не хранит свое имя. Имя хранится в каталоге. Поэтому жесткие ссылки могут давать одному inode несколько имен.

## Типы файлов

В выводе `ls -l` первый символ показывает тип файла.

| Символ | Тип | Смысл |
|---|---|---|
| `-` | регулярный файл | обычные данные: текст, бинарник, архив, изображение |
| `d` | каталог | таблица имен и inode |
| `l` | символическая ссылка | файл, содержащий путь к другому объекту |
| `s` | сокет | объект межпроцессного или сетевого взаимодействия |
| `b` | блочное устройство | устройство с блочным доступом, например диск |
| `c` | символьное устройство | потоковое устройство, например терминал |
| `p` | именованный канал | FIFO-канал между процессами |

### Регулярный файл

Регулярный файл - именованная область данных на носителе. В ext4 он представлен inode и блоками данных. При создании файла выделяется inode, при записи - блоки данных, а в inode появляются ссылки на эти блоки или экстенты.

Примеры:

```bash
touch notes.txt
echo "hello" > notes.txt
ls -li notes.txt
```

Ключ `-i` показывает номер inode.

### Каталог

Каталог - это специальный файл-таблица. Он хранит пары:

```text
имя -> номер inode
```

Из-за этого в одном каталоге нельзя создать два объекта с одинаковым именем. Каталоги формируют иерархию файловой системы.

Пример:

```bash
mkdir project
ls -ld project
```

### Жесткая ссылка

Жесткая ссылка - еще одно имя для того же inode. Команда:

```bash
ln file.txt file-hardlink.txt
```

После этого оба имени указывают на один inode. Данные исчезнут только тогда, когда счетчик жестких ссылок станет равен нулю и файл не будет открыт процессами.

Ограничения жестких ссылок:

- нельзя создавать жесткие ссылки на каталоги обычным способом, чтобы не ломать древовидную структуру;
- нельзя создавать жесткую ссылку между разными файловыми системами, потому что inode имеет смысл только внутри своей ФС.

### Символическая ссылка

Символическая ссылка - отдельный файл, внутри которого хранится путь к другому объекту:

```bash
ln -s /var/log/app.log app.log
```

Символическая ссылка может указывать на файл, каталог или даже на несуществующий путь. Если целевой объект удален, ссылка становится битой.

### Файлы устройств

Каталог `/dev` содержит специальные файлы устройств. Они не являются обычными данными на диске, а дают интерфейс к драйверам ядра.

Примеры:

- `/dev/sda`, `/dev/nvme0n1` - диски;
- `/dev/nvme0n1p1` - раздел;
- `/dev/tty` - текущий терминал;
- `/dev/null` - приемник, уничтожающий вывод;
- `/dev/zero` - источник нулевых байтов;
- `/dev/random` - источник случайных данных;
- `/dev/full` - устройство, возвращающее ошибку нехватки места.

## Атрибуты файлов

Основные атрибуты видны через `ls -l`:

```bash
ls -l /dev/null
```

Типичный вывод:

```text
crw-rw-rw- 1 root root 1, 3 May  9 12:00 /dev/null
```

Значимые поля:

- тип файла;
- права доступа;
- число жестких ссылок;
- владелец;
- группа;
- размер или major/minor для устройства;
- время изменения;
- имя.

Более подробные атрибуты показывает `stat`:

```bash
stat file.txt
```

Там важны:

- inode;
- размер;
- количество блоков;
- тип устройства;
- права в числовом и символьном виде;
- UID/GID владельца;
- время доступа `atime`;
- время изменения содержимого `mtime`;
- время изменения метаданных `ctime`;
- время рождения файла `birth`, если поддерживается ФС.

## Inode как центр метаданных

Индексный дескриптор хранит метаданные файла:

- тип файла;
- права доступа;
- UID владельца;
- GID группы;
- размер;
- временные метки;
- счетчик жестких ссылок;
- указатели на блоки данных или экстенты;
- служебные флаги файловой системы.

Имя файла в inode не хранится. Это объясняет поведение жестких ссылок и команд вроде `mv`: перемещение внутри одной файловой системы часто меняет только запись каталога, а не сами данные.

## Стандартные файловые дескрипторы процесса

При запуске обычный процесс получает три дескриптора:

| Номер | Имя | Назначение |
|---|---|---|
| `0` | stdin | стандартный ввод |
| `1` | stdout | стандартный вывод |
| `2` | stderr | стандартный поток ошибок |

Им соответствуют ссылки:

```text
/dev/stdin
/dev/stdout
/dev/stderr
```

Пример перенаправления:

```bash
command > out.txt 2> err.txt
command > /dev/null 2>&1
```

## Что сказать на экзамене коротко

Файл в Linux - универсальная абстракция доступа к данным и объектам ОС. Регулярные файлы, каталоги, ссылки, устройства, сокеты и каналы представлены в единой файловой иерархии. На дисковых ФС вроде ext4 файл описывается inode, а имя хранится в каталоге. Основные атрибуты: тип, права, владелец, группа, размер, inode, число ссылок и временные метки.

## Code

```bash
# типы и права файлов
ls -l

# inode файлов
ls -li

# подробные атрибуты
stat file.txt

# создать жесткую и символическую ссылки
ln file.txt hard.txt
ln -s file.txt soft.txt

# найти файлы с тем же inode в пределах ФС
find . -inum 12345

# посмотреть открытые файловые дескрипторы процесса
ls -l /proc/<PID>/fd
```
