## Zettel

**Главная мысль:** загрузка Linux поэтапна: прошивка находит загрузчик, загрузчик загружает ядро и `initramfs`, ядро инициализирует железо и через временную корневую ФС находит настоящий корень, после чего запускает процесс с PID 1.

## Развернутый ответ

Загрузка Linux решает проблему "курицы и яйца": чтобы читать диск, нужны драйверы и ядро, но чтобы загрузить ядро, сначала нужен минимальный код, который умеет найти его на носителе.

Общая последовательность:

1. BIOS или UEFI выполняет начальную инициализацию оборудования.
2. Прошивка находит загрузчик.
3. Загрузчик GRUB читает конфигурацию и позволяет выбрать ядро.
4. GRUB загружает `vmlinuz` и `initramfs` в память.
5. Ядро распаковывается и инициализирует подсистемы.
6. `initramfs` помогает найти и смонтировать настоящий корневой раздел.
7. Ядро запускает первый пользовательский процесс с PID 1.

## BIOS и MBR

BIOS - Basic Input/Output System. Это прошивка материнской платы, которая:

- выполняет POST;
- проверяет базовое оборудование;
- выбирает загрузочное устройство;
- передает управление загрузочному коду.

При BIOS-загрузке используется MBR - Master Boot Record. MBR занимает первые 512 байт диска:

- 440 байт - код загрузчика;
- 4 байта - сигнатура диска;
- 16 байт на каждый из четырех разделов;
- 2 байта - сигнатура `0x55AA`.

440 байт недостаточно для полноценного загрузчика, поэтому GRUB при BIOS-загрузке работает по фазам.

## GRUB при BIOS-загрузке

GRUB 2 - распространенный загрузчик Linux.

Фазы:

1. `boot.img` в MBR. Минимальный код, задача которого - найти следующий этап.
2. `core.img`. Обычно находится между MBR и первым разделом. Содержит базовые модули для чтения ФС, RAID, LVM.
3. Код и модули из `/boot/grub`. На этом этапе GRUB читает `grub.cfg`, показывает меню и загружает ядро.

`core.img` уже умеет читать файловые системы вроде ext4 или XFS, поэтому может найти `/boot/grub/grub.cfg`.

## UEFI и ESP

UEFI - более современная прошивка. Она обычно работает с GPT и умеет читать FAT16/FAT32, USB и сеть.

При UEFI-загрузке используется ESP - EFI System Partition. Это специальный раздел FAT, где лежат `.efi` загрузчики.

Пример структуры:

```text
/boot/EFI
├── ARCH/grubx64.efi
├── BOOT/BOOTX64.EFI
├── Microsoft/Boot/bootmgfw.efi
└── systemd/systemd-bootx64.efi
```

UEFI хранит загрузочные записи в NVRAM: каждая запись указывает на GUID раздела и путь к `.efi` файлу.

Преимущества UEFI:

- нет жесткого ограничения MBR;
- поддержка GPT;
- загрузчики хранятся как обычные файлы на ESP;
- возможна сетевая загрузка;
- возможен Secure Boot.

## Работа загрузчика

Загрузчик должен:

- найти доступные ядра;
- прочитать конфигурацию;
- дать выбрать запись загрузки;
- передать ядру параметры командной строки;
- загрузить в память ядро `vmlinuz`;
- загрузить `initramfs`;
- передать управление ядру.

Параметры ядра потом видны в:

```bash
cat /proc/cmdline
```

Типичные параметры:

```text
root=UUID=... rw quiet loglevel=3
```

`root=...` указывает настоящий корневой раздел.

## Загрузка ядра

`vmlinuz` - сжатый образ ядра Linux. После передачи управления ядро:

- распаковывается в памяти;
- настраивает защищенный режим процессора;
- инициализирует память и таблицы страниц;
- настраивает обработчики прерываний;
- запускает планировщик;
- обнаруживает устройства;
- подгружает встроенные драйверы;
- готовит запуск пользовательского пространства.

Но ядро не всегда может сразу смонтировать настоящий `/`. Например, для корня могут требоваться:

- драйвер контроллера диска;
- модуль файловой системы;
- LVM;
- RAID;
- расшифровка LUKS;
- сетевой root.

Для этого нужен стартовый виртуальный диск.

## `initramfs`

`initramfs` - Initial RAM Filesystem. Это архив, который загружается вместе с ядром, распаковывается в оперативную память и становится временным корнем.

Внутри обычно находятся:

- минимальные утилиты;
- модули ядра;
- скрипты ранней загрузки;
- инструменты для поиска root;
- средства для LVM, RAID, LUKS;
- программа `/init`.

Алгоритм:

1. Ядро распаковывает `initramfs`.
2. Запускается `/init` из временной ФС.
3. `/init` загружает модули и подготавливает устройства.
4. Находится настоящий корневой раздел.
5. Настоящий `/` монтируется.
6. Выполняется переключение корня.
7. Запускается обычная система.

Если root недоступен, система может попасть в аварийную shell внутри `initramfs`.

## `initrd` и `initramfs`

Раньше использовался `initrd` - Initial RAM Disk. Он представлялся как блочное устройство и монтировался как временная ФС.

`initramfs` отличается тем, что это архив, распаковываемый прямо в память как tmpfs-подобная корневая файловая система. В современных Linux чаще используется `initramfs`.

## PID 1

Когда настоящий root готов, ядро запускает первый пользовательский процесс:

```text
PID = 1
```

Исторически это был `/sbin/init`, в современных системах часто `/usr/lib/systemd/systemd` или `/sbin/init` как ссылка на systemd.

PID 1:

- запускает пользовательское пространство;
- является предком большинства процессов;
- усыновляет осиротевшие процессы;
- управляет завершением системы;
- переводит систему в рабочее состояние.

Если ядро не может запустить PID 1, обычно происходит kernel panic.

## Что сказать на экзамене коротко

Сначала BIOS/UEFI инициализирует оборудование и находит загрузчик. GRUB читает конфигурацию, показывает меню и загружает в память `vmlinuz` и `initramfs`. Ядро распаковывается, инициализирует подсистемы и запускает временный root из `initramfs`, чтобы найти настоящий корневой раздел. После переключения на настоящий root ядро запускает процесс с PID 1, например `init` или `systemd`.

## Code

```bash
# параметры, переданные ядру
cat /proc/cmdline

# файлы загрузки
ls -l /boot
ls -l /boot/grub
ls -l /boot/efi

# UEFI-записи
efibootmgr -v

# обновить конфигурацию GRUB в Debian/Ubuntu
sudo update-grub

# пересобрать initramfs в Debian/Ubuntu
sudo update-initramfs -u

# посмотреть PID 1
ps -p 1 -o pid,comm,args
```
