## Zettel

**Главная мысль:** файловая система превращает блочное устройство в управляемое дерево файлов; ext2 дала классическую inode-структуру, ext3 добавила журналирование, а ext4 улучшила масштабируемость, надежность и производительность.

## Развернутый ответ

**Файловая система** - способ организации информации на носителе данных. Она является посредником между пользователем, программами, системными вызовами и устройством хранения.

Основные функции файловой системы:

- организация хранения данных;
- учет и распределение свободного места;
- поддержка операций ввода-вывода;
- хранение метаданных;
- управление доступом;
- обеспечение целостности.

В Linux над конкретными файловыми системами работает слой **VFS** - Virtual File System. Он дает единый интерфейс системным вызовам (`open`, `read`, `write`, `close`, `stat`, `mount`) независимо от того, используется ext4, XFS, tmpfs, procfs или сетевая ФС.

## Способы организации данных

В лекции перечислены основные методы:

1. Линейное размещение.
2. Связный список.
3. Таблица аллокаций файлов.
4. Индексные дескрипторы.

Для Linux-системной ФС характерна модель с индексными дескрипторами. Файл описывается inode, а данные лежат в блоках или экстентах.

## Общая логика ext-семейства

Файловые системы ext2, ext3 и ext4 развивают одну линию:

- диск разбивается на блоки;
- блоки группируются в группы блоков;
- метаданные хранятся в суперблоке, дескрипторах групп, битовых картах и таблицах inode;
- каталоги связывают имена с inode;
- inode указывает на данные файла.

Такая структура удобна для локальности: inode, битовые карты и блоки данных в одной группе позволяют хранить связанные данные ближе друг к другу.

## ext2

**ext2** - Second Extended File System. Это классическая Linux-ФС без журналирования.

Особенности:

- использует inode;
- разделяет пространство на группы блоков;
- хранит суперблок, дескрипторы групп, битовые карты блоков и inode;
- проста и имеет небольшие накладные расходы;
- не ведет журнал операций.

Главный недостаток ext2 - восстановление после сбоя. Если питание пропало во время записи, ФС может оказаться в несогласованном состоянии. Тогда требуется проверка `fsck`, которая на больших разделах может занимать много времени.

Где ext2 может быть уместна:

- маленькие разделы;
- флеш-носители, где не хотят лишних записей журнала;
- специальные случаи, где важна простота.

Для современных системных разделов ext2 обычно не выбирают.

## ext3

**ext3** - развитие ext2 с добавлением журналирования. По структуре она близка к ext2, но имеет журнал, который помогает восстанавливать согласованность после сбоя.

Журналирование - это запись информации об операциях до окончательного применения изменений к основной структуре ФС. После аварии система может просмотреть журнал и довести операции до согласованного состояния или откатить незавершенные изменения.

В лекции выделены три режима журнала ext3:

| Режим | Смысл | Свойства |
|---|---|---|
| `journal` | в журнал пишутся и данные, и метаданные | самый надежный, но самый медленный |
| `ordered` | в журнал пишутся метаданные, а данные записываются до фиксации метаданных | баланс надежности и скорости |
| `writeback` | метаданные журналируются, порядок записи данных слабее | быстрее, но риск старых данных в файле после сбоя выше |

Преимущества ext3 перед ext2:

- быстрее восстановление после сбоя;
- меньше риск повреждения структуры ФС;
- совместимость с ext2 на уровне архитектуры.

Ограничения:

- производительность и масштабируемость хуже ext4;
- нет многих современных оптимизаций;
- метки времени менее точные.

## ext4

**ext4** - Fourth Extended File System, рекомендуемая в лекции ФС для Linux по умолчанию.

Ключевые улучшения ext4:

- **экстенты** вместо простой блочной адресации;
- поддержка больших файлов и больших файловых систем;
- отложенное выделение блоков;
- контрольные суммы журнала;
- дефрагментация без размонтирования;
- увеличение размера ФС во время работы;
- предварительное выделение места;
- метки времени с точностью до наносекунд;
- индексация больших каталогов через хеш-деревья.

### Экстенты

Экстент описывает непрерывный диапазон блоков. Вместо хранения множества ссылок на отдельные блоки inode может хранить описание:

```text
начальный логический блок файла -> начальный физический блок на диске, длина
```

Это уменьшает фрагментацию и ускоряет работу с большими файлами.

### Отложенное выделение блоков

В ext4 блоки могут выделяться не сразу при записи, а позже, когда система лучше понимает итоговый размер и характер записи. Это помогает выбрать более непрерывное место и уменьшить фрагментацию.

### Журнал и контрольные суммы

ext4 унаследовала журналирование от ext3 и добавила улучшения, включая контрольные суммы журнала. Это снижает риск некорректного восстановления при повреждении журнала.

## Сравнение ext2, ext3, ext4

| Свойство | ext2 | ext3 | ext4 |
|---|---|---|---|
| inode-структура | да | да | да |
| группы блоков | да | да | да |
| журналирование | нет | да | да |
| экстенты | нет | нет | да |
| быстрое восстановление после сбоя | нет | да | да |
| большие ФС и файлы | ограниченно | лучше ext2, но устаревше | да |
| точность времени до наносекунд | нет | нет | да |
| онлайн-дефрагментация | нет | нет | да |

## Разделы и FHS

Администрирование файловых систем связано с разделением диска. В лекции указана хорошая практика: выделять отдельные разделы для:

- EFI-раздела;
- `/boot`;
- корневого `/`;
- swap;
- `/home`;
- `/var`.

Это помогает изолировать разные типы данных. Например, переполненный `/var/log` не должен обязательно ломать домашние каталоги пользователей.

Стандарт FHS задает ожидаемые каталоги:

- `/bin`, `/sbin` - основные исполняемые файлы;
- `/etc` - конфигурация;
- `/dev` - устройства;
- `/proc`, `/sys` - виртуальные ФС;
- `/home` - домашние каталоги;
- `/var` - изменяемые данные;
- `/tmp` - временные файлы;
- `/usr` - пользовательские программы и общие данные.

## Что сказать на экзамене коротко

Файловая система организует хранение данных, метаданных, прав и свободного места. В Linux системные ФС часто основаны на inode. ext2 - простая inode-ФС без журнала; ext3 добавляет журналирование и ускоряет восстановление после сбоев; ext4 развивает ext3 за счет экстентов, отложенного выделения, контрольных сумм, больших размеров, онлайн-операций и более точных временных меток.

## Code

```bash
# посмотреть типы файловых систем
lsblk -f
findmnt
df -T

# создать ext4 на разделе
sudo mkfs.ext4 /dev/sdb1

# проверить ext-файловую систему
sudo fsck.ext4 /dev/sdb1

# посмотреть параметры ext-суперблока
sudo tune2fs -l /dev/sdb1

# узнать UUID файловой системы
blkid /dev/sdb1
```
