## Zettel

**Главная мысль:** System V init запускает систему последовательно по сценарию: PID 1 читает `/etc/inittab`, выполняет ранние команды, выбирает runlevel и запускает набор shell-скриптов из каталогов `rc*.d`.

## Развернутый ответ

System V init - классическая система инициализации Unix и старых Linux-дистрибутивов. Она работает после того, как ядро смонтировало корневую файловую систему и запустило первый пользовательский процесс с PID 1.

В модели SysV init:

- главным процессом является `init`;
- конфигурация обычно хранится в `/etc/inittab`;
- состояние системы описывается уровнем выполнения, runlevel;
- службы запускаются shell-скриптами;
- порядок запуска задается именами ссылок в `/etc/rc*.d/`.

## Роль PID 1

`init` с PID 1:

- запускает раннюю инициализацию;
- выбирает уровень выполнения;
- запускает системные службы;
- перезапускает важные процессы, например getty;
- принимает сигналы завершения и перезагрузки;
- усыновляет процессы без родителя.

Если `init` не запускается, система не переходит к нормальному пользовательскому пространству.

## `/etc/inittab`

Главный конфигурационный файл SysV init - `/etc/inittab`.

Формат строки:

```text
<id>:<runlevels>:<action>:<process>
```

Поля:

| Поле | Смысл |
|---|---|
| `id` | идентификатор записи |
| `runlevels` | уровни выполнения, где действует запись |
| `action` | условие или способ запуска |
| `process` | команда или скрипт |

Пример:

```text
id:3:initdefault:
si::sysinit:/etc/rc.d/rc.sysinit
l3:3:wait:/etc/rc.d/rc 3
1:2345:respawn:/sbin/getty 38400 tty1
```

## Действия в `/etc/inittab`

Типичные действия:

| Действие | Смысл |
|---|---|
| `initdefault` | уровень выполнения по умолчанию |
| `sysinit` | выполнить при ранней инициализации |
| `wait` | запустить и дождаться завершения |
| `respawn` | перезапускать после завершения |
| `once` | выполнить один раз при входе в runlevel |
| `boot` | выполнить при загрузке |
| `bootwait` | выполнить при загрузке и ждать |
| `ctrlaltdel` | действие на Ctrl+Alt+Del |
| `powerfail` | действие при проблемах питания |

В лекции отдельно выделены:

- `wait` - запустить и ждать завершения;
- `respawn` - перезапускать после завершения;
- `sysinit` - выполнить на раннем этапе загрузки.

## Уровни выполнения

Классические runlevels System V:

| Runlevel | Назначение |
|---|---|
| `0` | остановка системы |
| `1` | однопользовательский режим |
| `2` | многопользовательский режим без сети или ограниченный режим |
| `3` | многопользовательский текстовый режим |
| `4` | не используется или настраивается пользователем |
| `5` | многопользовательский графический режим |
| `6` | перезагрузка |

В Debian-подобных системах смысл отличался:

- `0` - остановка;
- `1` - recovery/single user;
- `2`, `3`, `4` - частичные многопользовательские режимы;
- `5` - полноценный режим;
- `6` - перезагрузка.

Важно: точный смысл runlevel может зависеть от дистрибутива.

## Скрипт `sysinit`

`sysinit` выполняется на раннем этапе загрузки до обычных служб. Он подготавливает систему:

- проверяет и монтирует файловые системы;
- настраивает `/proc`, `/sys`, `/dev`;
- активирует swap;
- устанавливает hostname;
- проверяет корневую ФС;
- очищает временные каталоги;
- подготавливает базовые устройства.

Пример записи:

```text
si::sysinit:/etc/rc.d/rc.sysinit
```

## Скрипты `rc` и каталоги `rc*.d`

Для каждого runlevel есть каталог:

```text
/etc/rc0.d/
/etc/rc1.d/
/etc/rc2.d/
/etc/rc3.d/
/etc/rc4.d/
/etc/rc5.d/
/etc/rc6.d/
```

В них обычно лежат символические ссылки на реальные скрипты из:

```text
/etc/init.d/
```

Имена ссылок задают действие и порядок:

```text
S20network
S80sshd
K10nginx
```

Где:

- `S` - start, запустить при входе в runlevel;
- `K` - kill, остановить при выходе или входе в runlevel;
- число задает порядок;
- оставшаяся часть имени указывает службу.

Скрипт службы обычно принимает аргументы:

```bash
/etc/init.d/sshd start
/etc/init.d/sshd stop
/etc/init.d/sshd restart
/etc/init.d/sshd status
```

## Последовательность загрузки SysV

Упрощенно:

1. Ядро запускает `/sbin/init`.
2. `init` читает `/etc/inittab`.
3. Выполняется `sysinit`.
4. Определяется runlevel по умолчанию.
5. Запускается `/etc/rc.d/rc <runlevel>` или аналогичный скрипт.
6. `rc` проходит по каталогу `/etc/rc<runlevel>.d/`.
7. Выполняются `K*` и `S*` ссылки в порядке номеров.
8. Запускаются терминалы `getty` через `respawn`.
9. Система готова к входу пользователей.

## Достоинства и недостатки System V

Плюсы:

- простая модель;
- shell-скрипты легко читать;
- порядок запуска явно виден по именам;
- минимум скрытой логики.

Минусы:

- последовательный запуск медленнее параллельного;
- зависимости выражаются слабо;
- обработка ошибок примитивна;
- сложнее автоматически перезапускать службы;
- нет единой модели для сокетов, таймеров, mount units.

## Что сказать на экзамене коротко

В System V после загрузки ядра запускается `/sbin/init` с PID 1. Он читает `/etc/inittab`, выполняет ранние команды с действием `sysinit`, выбирает runlevel и запускает скрипты через `rc`. Уровни выполнения определяют состояние системы: 0 - halt, 1 - single user, 3 - многопользовательский текстовый режим, 5 - графический, 6 - reboot. Службы запускаются скриптами из `/etc/init.d/` через ссылки `S*` и `K*` в `/etc/rc*.d/`.

## Code

```bash
# текущий runlevel
runlevel
who -r

# пример просмотра inittab
cat /etc/inittab

# скрипты служб
ls -l /etc/init.d/
ls -l /etc/rc3.d/

# управление службой в SysV-стиле
sudo service ssh start
sudo service ssh stop
sudo service ssh status

# ручной вызов init-скрипта
sudo /etc/init.d/ssh restart
```
