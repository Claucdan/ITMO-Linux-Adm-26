## Zettel

**Главная мысль:** доступ в Linux определяется не только битами `rwx`, но и тем, кем процесс является относительно файла: владельцем, участником группы или остальным пользователем; для каталогов значения `r`, `w`, `x` отличаются от значений для обычных файлов.

## Развернутый ответ

В Linux у каждого файла есть:

- владелец;
- группа;
- права для владельца;
- права для группы;
- права для остальных;
- специальные биты SUID, SGID и Sticky Bit.

Права видны в первом столбце `ls -l`:

```text
-rwxr-xr--
```

Первый символ - тип файла, следующие 9 символов - три тройки прав:

```text
u     g     o
rwx   r-x   r--
```

Где:

- `u` - user, владелец;
- `g` - group, группа;
- `o` - other, остальные.

## Значение прав для обычных файлов

| Право | Для файла |
|---|---|
| `r` | можно читать содержимое |
| `w` | можно изменять содержимое |
| `x` | можно запускать как программу или скрипт |

Пример:

```text
-rwxr-xr-- 1 alice developers 1200 May 9 deploy.sh
```

Расшифровка:

- `alice` может читать, писать и исполнять;
- группа `developers` может читать и исполнять;
- остальные могут только читать.

## Значение прав для каталогов

Для каталогов права имеют другой смысл.

| Право | Для каталога |
|---|---|
| `r` | можно читать список имен в каталоге |
| `w` | можно создавать, удалять и переименовывать записи каталога |
| `x` | можно проходить через каталог и обращаться к объектам внутри |

Важные комбинации:

- `r` без `x`: можно получить список имен, но нельзя нормально обращаться к файлам внутри;
- `x` без `r`: можно обратиться к известному имени, но нельзя посмотреть список каталога;
- `w` для каталога позволяет удалять файлы из каталога, даже если сам файл принадлежит другому пользователю, если нет Sticky Bit.

## Порядок проверки прав

Права проверяются слева направо по роли процесса относительно файла:

1. Если эффективный UID процесса равен владельцу файла, используются права владельца.
2. Иначе, если процесс входит в группу файла, используются права группы.
3. Иначе используются права остальных.

Если пользователь является владельцем, но у владельца нет прав, система не "добавит" права группы или остальных. Например:

```text
----r-xr-x
```

Владелец не сможет читать или исполнять файл, даже если группа и остальные могут.

## Числовая форма прав

Каждая тройка `rwx` кодируется восьмеричной цифрой:

| Бит | Значение |
|---|---|
| `r` | 4 |
| `w` | 2 |
| `x` | 1 |

Примеры:

| Число | Права |
|---|---|
| `7` | `rwx` |
| `6` | `rw-` |
| `5` | `r-x` |
| `4` | `r--` |
| `0` | `---` |

Полные примеры:

```bash
chmod 755 script.sh   # rwxr-xr-x
chmod 644 file.txt    # rw-r--r--
chmod 700 ~/.ssh      # rwx------
chmod 600 id_rsa      # rw-------
```

## Символьная форма прав

Команда `chmod` может менять права через буквенные выражения:

```bash
chmod u+x script.sh
chmod g-w report.txt
chmod o= project.txt
chmod u=rw,g=r,o= file.txt
chmod u+w,o-w file.txt
```

Обозначения:

- `u` - владелец;
- `g` - группа;
- `o` - остальные;
- `a` - все;
- `+` - добавить;
- `-` - убрать;
- `=` - установить ровно указанные права.

## Специальные биты

Помимо 9 основных битов есть 3 специальных.

### SUID

SUID на исполняемом файле означает: процесс запускается с эффективными правами владельца файла.

Обозначение:

```text
-rwsr-xr-x
```

Если `x` у владельца не установлен:

```text
-rwSr-xr-x
```

Типичный смысл: обычный пользователь запускает программу, которой временно нужны права владельца, например `root`.

Установка:

```bash
chmod u+s program
chmod 4755 program
```

### SGID

SGID на исполняемом файле означает запуск с эффективной группой файла.

На каталоге SGID означает: новые файлы и подкаталоги внутри наследуют группу каталога, а не основную группу пользователя. Это полезно для общих рабочих директорий.

Установка:

```bash
chmod g+s shared/
chmod 2775 shared/
```

### Sticky Bit

Sticky Bit для каталога означает: файл внутри каталога может удалить или переименовать только владелец файла, владелец каталога или `root`.

Классический пример - `/tmp`:

```text
drwxrwxrwt
```

Установка:

```bash
chmod +t /shared/tmp
chmod 1777 /shared/tmp
```

## Владельцы и группы

Сменить владельца:

```bash
chown alice file.txt
```

Сменить владельца и группу:

```bash
chown alice:developers file.txt
```

Сменить группу:

```bash
chgrp developers file.txt
```

Рекурсивное изменение:

```bash
chown -R alice:developers /srv/project
chmod -R g+rwX /srv/project
```

Здесь `X` добавляет исполнение только каталогам и тем файлам, у которых уже был установлен исполняемый бит.

## umask

`umask` задает маску прав, которые будут сняты при создании новых файлов и каталогов.

Обычно:

- файл создается максимум как `666`;
- каталог создается максимум как `777`;
- затем из них вычитается `umask`.

Пример:

```bash
umask 022
```

Итог:

- новый файл: `644`;
- новый каталог: `755`.

Для приватной среды часто используют:

```bash
umask 077
```

Итог:

- файл: `600`;
- каталог: `700`.

## Что сказать на экзамене коротко

Права доступа в Linux основаны на владельце, группе и остальных пользователях. Для каждой категории задаются биты `r`, `w`, `x`; для файлов они означают чтение, запись и исполнение, а для каталогов - просмотр имен, изменение записей и проход по пути. Управление выполняется через `chmod`, `chown`, `chgrp`, а специальные биты SUID, SGID и Sticky Bit задают особое поведение запуска и удаления.

## Code

```bash
# посмотреть права
ls -l
stat file.txt

# изменить права
chmod 640 file.txt
chmod u+x script.sh
chmod g+s shared/
chmod +t shared-tmp/

# изменить владельца и группу
chown root:root file.txt
chgrp developers file.txt

# проверить umask
umask
umask 027
```
