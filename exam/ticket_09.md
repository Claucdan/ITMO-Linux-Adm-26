# Билет 9. Отображение структур данных о процессе в псевдофайловую систему `/proc`

## PARA

- **Project:** подготовка к экзамену по администрированию Linux.
- **Area:** процессы, ядро, псевдофайловые системы, диагностика.
- **Resources:** `lectures/admlinux_2026_03_20.md`, `lectures/admlinux_2026_03_06.md`.
- **Archive:** связанный материал: [Билет 7](./ticket_07.md), [Билет 8](./ticket_08.md), [Билет 10](./ticket_10.md).

## Zettel

**Главная мысль:** `/proc` - это не каталог с обычными файлами на диске, а интерфейс ядра, который отображает внутренние структуры процессов и параметры системы в виде файлов и каталогов.

## Развернутый ответ

`/proc` - псевдофайловая система, через которую пользовательские программы и администратор получают сведения о процессах и ядре. Данные в `/proc` не читаются с диска: при обращении к файлу ядро формирует содержимое из своих структур в оперативной памяти.

Для процессов это особенно важно: структура ядра `task_struct`, таблица открытых файлов, сведения о памяти, планировщике и окружении отображаются в дерево:

```text
/proc/<PID>/
```

Где `<PID>` - идентификатор процесса.

## Почему `/proc` является псевдофайловой системой

Обычная файловая система хранит данные на носителе: ext4, XFS, Btrfs. `/proc` хранит представление состояния ядра.

Отличия:

- файлы часто имеют нулевой размер в `ls`, но при чтении отдают данные;
- содержимое меняется динамически;
- многие файлы существуют только пока существует процесс;
- чтение файла может быть операцией запроса к ядру;
- некоторые файлы доступны только root или владельцу процесса.

Проверить тип:

```bash
findmnt /proc
mount | grep proc
```

## Каталоги процессов

Для каждого процесса создается каталог:

```text
/proc/1
/proc/1234
/proc/<PID>
```

Когда процесс завершается и его структуры удаляются из таблицы процессов, каталог исчезает.

Важные элементы `/proc/<PID>`:

| Путь | Назначение |
|---|---|
| `/proc/<PID>/cmdline` | командная строка запуска |
| `/proc/<PID>/exe` | ссылка на исполняемый файл |
| `/proc/<PID>/cwd` | текущий рабочий каталог |
| `/proc/<PID>/environ` | переменные окружения |
| `/proc/<PID>/status` | человекочитаемая сводка о процессе |
| `/proc/<PID>/stat` | машинно-читаемая статистика процесса |
| `/proc/<PID>/io` | статистика ввода-вывода |
| `/proc/<PID>/maps` | карта виртуальной памяти |
| `/proc/<PID>/pagemap` | сведения о страницах памяти |
| `/proc/<PID>/statm` | размеры памяти процесса |
| `/proc/<PID>/sched` | статистика планировщика |
| `/proc/<PID>/fd` | открытые файловые дескрипторы |

## Командная строка и исполняемый файл

`cmdline` содержит аргументы запуска процесса. Аргументы разделены нулевыми байтами, поэтому для удобного просмотра используют `tr`:

```bash
tr '\0' ' ' < /proc/<PID>/cmdline
```

`exe` - символическая ссылка на реальный исполняемый файл:

```bash
readlink -f /proc/<PID>/exe
```

Это полезно, когда команда запуска выглядит как `python app.py`, `bash script.sh` или `nano test.txt`: `exe` покажет интерпретатор или исполняемый файл, реально загруженный процессом.

## Текущий каталог и окружение

`cwd` показывает рабочий каталог процесса:

```bash
readlink -f /proc/<PID>/cwd
```

`environ` содержит переменные окружения, тоже разделенные нулевыми байтами:

```bash
tr '\0' '\n' < /proc/<PID>/environ
```

Переменные окружения могут содержать секреты, поэтому доступ к ним ограничивается правами.

## `/proc/<PID>/status`

`status` - удобная человекочитаемая сводка. В ней можно найти:

- имя процесса;
- состояние;
- PID и PPID;
- UID и GID;
- количество потоков;
- сведения о памяти;
- маски сигналов;
- capabilities;
- привязку к cgroup и namespace в современных системах.

Пример:

```bash
cat /proc/<PID>/status
```

Особенно важные поля:

| Поле | Смысл |
|---|---|
| `Name` | имя процесса |
| `State` | состояние: running, sleeping, zombie и т.д. |
| `Pid` | PID |
| `PPid` | PID родителя |
| `Uid` | реальные и эффективные UID |
| `Gid` | реальные и эффективные GID |
| `Threads` | число потоков |
| `VmSize` | виртуальный размер памяти |
| `VmRSS` | резидентная память в RAM |

## `/proc/<PID>/stat` и `statm`

`stat` предназначен для машинной обработки. Его используют `ps`, `top` и другие утилиты. Там в одной строке находится много числовых полей: PID, имя, состояние, PPID, времена CPU, приоритеты, адреса и т.д.

`statm` содержит 7 чисел о памяти процесса:

```text
size resident shared text lib data dt
```

Из лекции:

- общий размер программы;
- резидентная часть в ОЗУ;
- разделяемая память;
- сегмент кода;
- библиотеки, поле устарело;
- данные и стек;
- грязные страницы, поле устарело.

## Карта памяти: `maps` и `pagemap`

`maps` показывает области виртуальной памяти:

```bash
cat /proc/<PID>/maps
```

Там видны:

- адресные диапазоны;
- права на область: `r`, `w`, `x`, `p`/`s`;
- смещение в файле;
- устройство и inode;
- путь к отображенному файлу, библиотеке, heap или stack.

Это помогает понять:

- какие библиотеки загружены;
- где находится стек;
- где heap;
- какие файлы отображены через `mmap`.

`pagemap` содержит низкоуровневые 64-битные записи о страницах и обычно требует повышенных прав.

## Открытые файловые дескрипторы

Каталог `/proc/<PID>/fd` содержит символические ссылки на открытые дескрипторы процесса:

```bash
ls -l /proc/<PID>/fd
```

Пример:

```text
0 -> /dev/null
1 -> /var/log/app.log
2 -> /var/log/app.err
3 -> socket:[12345]
```

Это один из главных инструментов диагностики:

- какой файл держит процесс;
- почему нельзя размонтировать файловую систему;
- куда демон пишет логи;
- какие сокеты открыты.

## Сведения о вводе-выводе и планировщике

`/proc/<PID>/io` показывает статистику операций с хранилищем:

```text
rchar
wchar
syscr
syscw
read_bytes
write_bytes
cancelled_write_bytes
```

`/proc/<PID>/sched` содержит данные планировщика: времена выполнения, переключения контекста, параметры планирования.

## Общесистемные файлы `/proc`

Кроме `/proc/<PID>`, есть файлы о системе:

| Путь | Назначение |
|---|---|
| `/proc/cmdline` | параметры запуска ядра |
| `/proc/cpuinfo` | сведения о CPU |
| `/proc/meminfo` | сведения об оперативной памяти |
| `/proc/diskstats` | статистика дисков |
| `/proc/devices` | список устройств |
| `/proc/mounts` | смонтированные ФС |
| `/proc/modules` | загруженные модули ядра |
| `/proc/filesystems` | поддерживаемые ФС |
| `/proc/swaps` | активная подкачка |
| `/proc/version` | версия ядра |
| `/proc/sys/kernel` | изменяемые параметры ядра |

Некоторые параметры в `/proc/sys` можно менять на лету. Например:

```bash
cat /proc/sys/kernel/hostname
```

Более правильный административный способ для постоянных настроек - `sysctl` и файлы в `/etc/sysctl.d/`.

## Что сказать на экзамене коротко

`/proc` отображает внутренние структуры ядра в файловый интерфейс. Для каждого процесса есть каталог `/proc/<PID>`, где можно посмотреть командную строку, исполняемый файл, рабочий каталог, окружение, состояние, память, статистику ввода-вывода, планировщик и открытые файловые дескрипторы. Эти файлы не лежат на диске: ядро формирует их динамически при чтении.

## Code

```bash
# информация о процессе
cat /proc/<PID>/status
tr '\0' ' ' < /proc/<PID>/cmdline
readlink -f /proc/<PID>/exe
readlink -f /proc/<PID>/cwd

# дескрипторы и память
ls -l /proc/<PID>/fd
cat /proc/<PID>/maps
cat /proc/<PID>/io
cat /proc/<PID>/sched

# общая информация о системе
cat /proc/cmdline
cat /proc/cpuinfo
cat /proc/meminfo
cat /proc/mounts
cat /proc/modules
```
